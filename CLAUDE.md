# パチスロ判定アプリ（working title: pachi-judge）

## プロジェクト概要
スマホのカメラでパチスロ台のデータカウンター（ゲーム数・大当たり回数等）を撮影し、機種ごとの攻略データ（天井・ゾーン・独自pt等）と組み合わせて「打つべきか(GO)／様子見(STAY)／やめるべきか(STOP)」を信号機UIで判定するPWA。設定差（機械割の設定1〜6差）は判定ロジックに使わない（MVP方針）。

## 技術スタック
- フロントエンド: Next.js（PWA化）, TypeScript, Tailwind CSS
- ホスティング: Vercel（フロントエンド＋サーバーレス関数を1か所で）
- データベース: Google スプレッドシート（Google Sheets API、サービスアカウント認証で読み取り）
- 画像解析・判定AI: Anthropic Claude API（vision入力で機種名・現在の数値を読み取り、判定計算もJSON形式を強制して出力させる）

## アーキテクチャ（処理の流れ）
1. ユーザーがPWAで台を撮影
2. Vercelのサーバーレス関数が画像を受け取る
3. Claude APIに画像を渡し、機種名と現在の数値（G数／独自カウンター等）を読み取らせる
4. 読み取った機種名でGoogleスプレッドシートの「機種マスタ」「機種ルール詳細」を検索し、該当行を取得する
5. 取得したスペックデータ（正しい数値）をプロンプトに埋め込み、Claude APIに判定させる。**機種ルール詳細に無い数値はAI自身の記憶で補わせない**
6. 判定結果（GO/STAY/STOP＋理由＋参照ルール＋推定投資額＋次の確認タイミング）をJSON形式で受け取り、信号機UIに表示する

## 判定ロジックの大原則
- 「場面A：着席前判定」と「場面B：やめ時判定」を必ず分けて考える
- 場面A：ゾーン・天井までの距離と投資見積もりからGO/STAY/STOPを出す
- 場面B：機種ルール詳細の「やめ時」行にある"即ヤメ厳禁の例外条件"への合致を最優先で確認する
- 1機種に複数の天井・ゾーンがある場合は、現在地から最も近いものを優先する
- 詳細なロジックの正本はGoogleスプレッドシートの「判定ロジック」シート。コードにロジックをハードコードせず、プロンプトテンプレート経由でスプレッドシートの内容を都度反映させる

## データソースに関する重要な制約
- 機種データは自動スクレイピングしない。半自動キュレーション（人がサイトを確認→AIが抽出→人が承認→シートに反映）で更新する運用が前提
- 機種データがスプレッドシートに未登録の場合、AIに数値を推測させず「データ未登録」として保守的にSTAY/STOPへ倒す
